- the join command should be moved to the collector cli
**It should be moved to the collector CLI.** The server binary should only contain `serve`, `generate-token`, and `init`. The `join` command, along with its helper functions (`parse_server_url`, `fetch_server_cert_fingerprint`, `pair_collector`, `der_to_pem`, `sha256_fingerprint`, `InsecureServerCertVerifier`, etc.) belong in the collector binary since that's the component being enrolled.


- why are things such as LIFELOG_RETENTION_INTERVAL_SECS an environment variable? instead, it should be a part of the server's config. here is how i want the config to be. the server should have access to the lifelog-config.toml file. this file should have all of the server's configuration. each collector (device) should have its own lifelog-config.toml file that contains its configuration. Howeveer, the same data model should be used for all configs on all devices to simplify the program.
- othert hings like lifelog_host, lifelog_port, etc. should all be coming from the config, instead of being passed in as env. make a note of this for the proejct, all configuration should be coming from the config file, and not from env variables. The only thing that should be coming from env variables are secrets such as database passwords, api keys, etc.


- by default, we do not have a retention interval. we store everything for forever, but we can configurate it. retention should be measured in days/weeks/months/years and not seconds.
